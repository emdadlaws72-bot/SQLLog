# MySQL 实时SQL监控小工具 V2.0

## 📖 产品简介

**仓储现场开发/测试/定位问题专用工具**

无需部署，无需安装依赖，开箱即用，实时监控 MySQL 所有执行的 SQL 语句。

---

## 🚀 快速开始

### 方式一：直接运行（推荐）

1. 双击 `启动工具.bat`
2. 等待自动打开浏览器（如未打开，手动访问 http://localhost:3003）
3. 输入数据库连接信息
4. 点击"连接数据库"
5. 点击"开始监控"

### 方式二：Node.js 运行

```bash
npm install
npm start
```

## 📋 功能特性

### ✅ 核心功能

| 功能 | 说明 |
|------|------|
| 实时 SQL 监控 | 每秒抓取 performance_schema 的 SQL 执行记录 |
| 自动保存 | 每 2000 条 SQL 自动保存为 TXT 文件并清空 |
| 停止保存 | 停止监控时自动保存当前数据 |
| 历史日志查看 | 查看所有历史保存的 SQL 日志文件 |

### ✅ 筛选功能

- **SQL 类型筛选**：SELECT/INSERT/UPDATE/DELETE/全部
- **执行时间筛选**：只显示执行时间≥配置值的 SQL
- **业务类型筛选**：入库/出库/补货/盘点/空容器/库存管理
- **数据库筛选**：连接后自动获取数据库列表
- **表名筛选**：选择数据库后自动获取表名
- **涉及表筛选**：实时识别 SQL 中涉及的表

## ⚙️ 数据库配置要求

### 1. 确保 performance_schema 已启用(mysql5.7版本以上是默认开启的 包含5.7)

在 MySQL 中执行：
```sql
SHOW VARIABLES LIKE 'performance_schema';
```

如果显示 `ON`，说明已启用。如果是 `OFF`，需要在 `my.cnf` 或 `my.ini` 中添加：
```ini
[mysqld]
performance_schema = ON
```

### 2. 确保 events_statements_history 已启用

```sql
SHOW VARIABLES LIKE 'performance_schema_events_statements_history_size';
```

建议设置为 10000 或更大：(默认是1024 ,这个值代表sql文件大小 如果字节大于这个就不监控)
```sql
SET GLOBAL performance_schema_events_statements_history_size = 10000;
```

## 🔧 配置说明

### 连接配置

- **主机地址**：MySQL 服务器地址（默认 localhost）
- **端口**：MySQL 端口（默认 3306）
- **用户名**：数据库用户名（需要有访问 performance_schema 的权限）
- **密码**：数据库密码
- **保存路径**：SQL 日志保存路径（默认根目录 可以自己配）

### 监控配置

- **执行时间筛选（秒）**：只记录执行时间≥此值的 SQL，0 表示不限制
- **SQL 类型筛选**：只记录指定类型的 SQL
- **保存路径**：自动保存文件的目录

## ⚠️ 注意事项

1. **权限要求**：数据库用户需要有访问 `performance_schema` 的权限
2. **性能影响**：监控会轻微影响数据库性能，生产环境慎用
3. **数据保存**：每 2000 条自动保存，建议定期清理旧文件

## 🔧 常见问题

### Q: 无法连接数据库？
A: 请检查：
- 数据库地址、端口是否正确
- 用户名密码是否正确
- 是否有访问 performance_schema 的权限
- 防火墙是否开放

### Q: 看不到 SQL 记录？
A: 请检查：
- performance_schema 是否已启用
- 是否有正在执行的 SQL
- 筛选条件是否太严格

### Q: 配置没有保存？
A: 配置会在连接数据库时自动保存到 config.json
